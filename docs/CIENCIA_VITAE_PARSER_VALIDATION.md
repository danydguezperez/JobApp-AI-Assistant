## Validation Conclusion

The local desktop parser is useful for job-application matching and can recover many important Ciência Vitae fields without AI credits. However, it is not equivalent to the curated HTML generated by Claude:

- The curated HTML has richer education, project, conference, dataset, teaching, and service organization.
- The local heuristic combines several output types into broader buckets.
- Some local rows are snippets rather than fully normalized records.
- The Web Lite parser is more limited because it runs only in the browser and cannot use the full Python/FastAPI parser or local enrichment logic.

## Product Positioning

Public copy should say:

- Desktop/local app: recommended for real Ciência Vitae parsing, sensitive CVs, long workflows, local exports, and repeated applications.
- Web Lite: experimental browser version for quick tests, BYOK learning, and a first heuristic pass over PDF/TXT/Markdown/JSON/XML CVs.
- Curated publication-ready CV pages, like the reference HTML, still require a higher-quality AI/manual curation layer.

This supports the current d'BiYOK Lab messaging: practical, transparent, local-first, and honest about what each mode can and cannot do.
